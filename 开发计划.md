## Phase 3：后续迭代 — 安全加固 ⬜ 待做

> 目标：达到可上线的安全标准

### 3.1 密码逻辑服务端化

**方案：** 使用 Supabase Edge Functions（Deno 运行时）

- [ ] 创建 Edge Function `login`：接收邮箱+密码，服务端查表 + bcrypt 比对，返回用户信息
- [ ] 创建 Edge Function `register`：接收注册信息，服务端查重 + bcrypt 加密 + 插入记录
- [ ] 创建 Edge Function `reset-password`：发送邮件验证码 + 重置密码
- [ ] 前端移除 bcryptjs 依赖，改为调用 Edge Function

### 3.2 后端中间层

- [ ] 所有数据库写操作（提交活动、兑换积分、更新资料）通过 Edge Function 代理
- [ ] Edge Function 中增加权限校验（检查 is_verified、user_type）
- [ ] 积分兑换增加并发控制（防止超卖）
- [ ] 文件上传增加类型和大小校验

### 3.3 Supabase 安全配置

- [ ] 配置 RLS（Row Level Security）策略：
  - `users` 表：anon 只能读自己的数据（通过 auth.uid 关联）
  - `activity_submissions` 表：只能插入自己的记录
  - `points_exchange_applications` 表：只能查看和插入自己的记录
  - 其他只读表：anon 可读，只有 service_role 可写
- [ ] anon key 权限最小化
- [ ] 配置 Supabase Auth（邮箱验证流程）

### 3.4 认证系统迁移到 Supabase Auth（方案 A）

- [ ] Supabase 控制台开启 Email provider
- [ ] 配置 SMTP（QQ/163 邮箱授权码）
- [ ] 重写 Register：用 `supabase.auth.signUp()` 注册
- [ ] 重写 Login：用 `supabase.auth.signInWithPassword()` 登录
- [ ] 重写 ForgotPassword：用 `supabase.auth.resetPasswordForEmail()` 发邮件
- [ ] 新建重置密码页面（邮件链接跳转落地页）
- [ ] userStore 改为监听 `supabase.auth.onAuthStateChange()`
- [ ] 移除前端 bcryptjs 依赖

### 3.5 输入安全

- [ ] 所有用户输入做 XSS 过滤（DOMPurify 或 sanitize-html）
- [ ] Markdown 渲染时过滤危险标签和属性
- [ ] 图片上传限制文件类型（jpg/png/webp）和大小（< 5MB）
